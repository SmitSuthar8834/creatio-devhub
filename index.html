What it does

Everything clio can do, without living in the terminal.

DevHub drives the tools you already have — clio, Git, and the GitHub CLI — and represents every long operation as a streamed, cancellable job.

Environments

Register & manage

Add Creatio environments, ping them, set a default, and install the cliogate helper — credentials stay in clio, never in DevHub.

Workspaces & Git

Source control, visual

Create empty or full workspaces, add packages, review colorized diffs, commit, and inspect history — GitHub Desktop, for Creatio packages.

GitHub, both ways

Publish & restore

Create a GitHub repo straight from a workspace and push it — or deploy a package back from a repo to restore a broken environment.

Deployment

Env → env, safely

Move a package or a whole application between environments, with a drift guard that blocks pushing over changes you haven't pulled.

Jobs

Watch everything run

Streamed logs, a top-right activity indicator, safe cancellation of process trees, and a persistent job history across restarts.

SQL

Query, inspect & export

Run SQL against a Creatio environment, inspect results in a fast grid, and export the complete result to CSV or Excel.

Updates

Signed & automatic

Cryptographically verified releases from GitHub. DevHub checks, downloads, verifies, and installs — no manual reinstalls.

The loop

Pull, edit, version, ship — a real workflow.

01

Pull

Bring editable packages from an environment into a local workspace.

02

Commit

Diff the changes and commit them to Git — history you can trust.

03

Push to GitHub

Create the repo from DevHub and push, private by default.

04

Deploy

Install to the next environment — with a backup and drift check.

Changelog

What's new

DevHub ships often, and updates install themselves after the first download.

v0.2.8latest
July 2026

Deployment errors you can act on

Failed jobs now explain the outcome in plain language and separately preserve the real technical error. Partial package deployments identify skipped schemas and local-change conflicts instead of ending with only a generic error.

v0.2.7
July 2026

SQL runner & CSV/Excel export

Choose an environment, run SQL from a keyboard-friendly editor, inspect up to 2,000 rows in a sticky, scrollable grid, and export the complete result directly to CSV or Excel.

v0.2.6
July 2026

Deploy from GitHub

Clone a repository at a chosen branch and install it straight into an environment — restore a broken environment from known-good source, or move a package onto a fresh one. Repo & branch picker, backup, and keep-as-workspace options.

v0.2.5
July 2026

Live job indicator & auto-captured catalog

A top-right indicator shows running jobs from any screen and flashes each outcome. Package and application lists are now captured into the cache automatically on launch and when you switch environments.

v0.2.4
July 2026

Empty-first workspaces & GitHub from the app

Start an empty workspace and add only the packages you want, then create a private GitHub repo and push — without leaving DevHub.

v0.2.3
July 2026

New look & persistent job history

Refreshed branding and a light sidebar, plus a job history that survives restarts so you can revisit past runs and their logs.

v0.2.1

First public release

Signed Windows installers (NSIS & MSI) published to GitHub, with cryptographically verified auto-updates.

Requirements

  • ▹Windows 10 or 11 (x64). A small, signed native installer.
  • ▹The clio CLI, Git, and the GitHub CLI (gh) installed — DevHub drives them.
  • ▹A registered Creatio environment (cloud or on-site) to connect to.

Built for trust

  • ▹No credentials stored. clio owns your environment secrets — DevHub never holds them.
  • ▹Every mutation is a job. Streamed output, serialized per environment, safe to cancel.
  • ▹Signed updates. Releases are verified against the project key before install.
